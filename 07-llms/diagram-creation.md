# Diagram Creation Inside LLMs

Best practices for generating diagrams using AI tools. "Inside LLMs" means the LLM generates the diagram artifact (SVG, Mermaid, DOT, etc.) — not a separate tool invoked by the LLM.

## Format choices

**Mermaid** — the most portable. Renders natively in Obsidian, GitHub, and many markdown viewers. Best for flowcharts, sequence diagrams, Gantt charts, and simple network diagrams. Syntax is declarative, which means LLMs generate it reliably because there's less room for structural error than in SVG.

**SVG** — more expressive but error-prone when generated by LLMs. Fine for simple geometric shapes; complex diagrams with precise positioning often require iteration. Claude Code's `show_widget` tool renders SVG inline — useful for visual validation during a session.

**ASCII / box-drawing** — works in any terminal or plain-text file. Low fidelity, but sometimes the right call for a quick communication diagram in a document.

## Claude

Claude Code can render SVG inline via the `show_widget` call and generate Mermaid for Obsidian embeds. For process flow diagrams (like decoking sequence flows), Mermaid is the preferred output because it renders in the vault and is editable as text. SVG is better when precise layout control matters.

## Tips

Provide explicit node labels and a rough layout description in the prompt. Asking for "a diagram of X" without specifying node names, connection direction, and layout orientation produces generic results that need heavy revision. Specify the target renderer (Obsidian, GitHub, etc.) so the LLM can pick compatible syntax.
